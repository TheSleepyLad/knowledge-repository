# 白名单合并流程

更新时间：2026-07-07

用途：你和 Codex 分别收集候选账号后，按同一套流程合并为最终白名单。

## 文件分工

| 文件 | 用途 |
|---|---|
| `候选视频博主白名单.md` | 初步候选池，允许信息不完整 |
| `分学科候选账号收集表.md` | 按小学、初中、生活通识与学科分组收集 |
| `白名单审核评分标准.md` | 统一评分、风险标签和一票否决规则 |
| `正式白名单.md` | 审核通过，可进入稳定采集 |
| `试采白名单.md` | 可小规模采集，需要继续人工抽检 |
| `排除清单.md` | 明确不采集，记录原因，避免重复评估 |

## 合并步骤

1. 收集候选
   - 你和 Codex 各自补充候选账号。
   - 每个账号至少写清：平台、账号名、账号主页、学段、学科、推荐理由。

2. 去重归并
   - 同名账号、跨平台同主体账号合并为一条。
   - 如果同一主体在 B站、抖音、官网均有内容，优先官网或 B站作为主来源，抖音作为辅助发现来源。

3. 样本抽检
   - 每个账号抽 10 条公开视频。
   - 覆盖最近视频、热门视频、低播放视频、不同知识点。
   - 记录样本链接、知识点、适龄性、广告风险、事实错误。

4. 打分
   - 使用 `白名单审核评分标准.md`。
   - 先检查一票否决项，再打 100 分制。

5. 定级
   - A 正式白名单：>= 85 分。
   - B 试采白名单：70-84 分。
   - C 候选观察：55-69 分。
   - D 排除：< 55 分或触发一票否决。

6. 采集策略确认
   - 正式白名单也默认只采集：标题、链接、发布时间、账号、摘要、知识点标签、人工审核状态。
   - 不默认采集完整字幕、完整正文、视频文件、付费资料、讲义 PDF。

7. 定期复核
   - 每月复核一次正式白名单账号是否出现营销化、内容漂移、停更、争议或平台限制变化。

## 合并冲突处理

| 冲突 | 处理方式 |
|---|---|
| 你给高分，Codex 给低分 | 先按低分进入试采或候选，补充样本后复评 |
| 一方认为触发一票否决 | 进入人工复核，不直接进入正式白名单 |
| 账号内容质量高但版权不清 | 只能采元数据和链接，不采摘要外的长文本 |
| 抖音账号优质但碎片化 | 只作为发现来源，尽量找其官网/B站/公众号同源内容 |
| 科普账号优质但超纲 | 进入生活通识或拓展材料，不进入核心学科知识点 |
